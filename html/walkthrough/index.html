2. Responsive Grid Layout Bootstrap uses a 12-column grid system. To make it work, you must follow this hierarchy: Container > Row > Column. Plain-English Explanation: Container: Centers your content and adds side padding so it doesn't touch the screen edges. Row: A horizontal wrapper for your columns. Col: The actual content holders. We use breakpoints like md (medium) to tell the columns when to stack or sit side-by-side.

3. Adding a Responsive Navbar The Navbar is one of Bootstrap's most important components. It automatically collapses into a hamburger menu on mobile devices. Plain-English Explanation: The navbar-expand-lg class keeps the menu open on large screens. On smaller screens, the navbar-toggler button appears. The data-bs-target must match the id of the collapsible div for the menu to open.

Summary: 5 Things to Remember Mobile-First Design: Bootstrap is designed for small screens first; use breakpoints like md or lg to adjust layouts for larger screens. The 12-Column Rule: Rows are divided into 12 units. Two col-6 elements will fill exactly one row. Hierarchy is Key: Always wrap your grid items in this order: Container > Row > Col. JS is Mandatory: Interactive components like the navbar hamburger menu or modals will not work without the Bootstrap JS Bundle link. Utility Classes: Use classes like p-4 (padding) or mt-3 (margin) to handle spacing quickly instead of writing custom CSS.
